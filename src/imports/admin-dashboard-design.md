Design an enterprise-grade Admin Dashboard UI for a large-scale content publishing system that manages RSS feeds, article ingestion, editing, and review workflows.

The product should feel like it belongs to a multinational SaaS company — similar in quality and clarity to platforms like Stripe Dashboard, HubSpot CMS, or Webflow Enterprise.

Focus on:
• Clean enterprise layout
• Strong information hierarchy
• Clear workflow-based navigation
• Scalability for thousands of articles
• Multi-admin usage
• Role-based access structure
• Performance-focused UX

Core Modules to Design:

1. Global Layout System

* Left collapsible sidebar navigation
* Top header with search, notifications, profile dropdown
* Breadcrumb navigation
* Page title + contextual actions
* Right-side contextual panel (optional for details)

2. Dashboard (Admin Overview)

* Stats cards (Total Feeds, Articles Today, Pending Reviews, Published)
* Recent Activity Timeline
* System Health Indicator
* Quick Actions section

3. RSS Feed Management
   A. RSS Feed Form

* Structured card-based form layout
* Input fields with labels + helper text
* Validation states
* Frequency selector
* Toggle switches
* Save & Test Feed button
* Confirmation modal

B. RSS Feed List

* Enterprise data table
* Search, advanced filters, sorting
* Bulk select
* Status chips (Active, Paused)
* Pagination
* Empty state with illustration

4. Article Queue

* Hybrid layout (table + expandable rows)
* Article metadata preview
* Status badges (Draft / Needs Review / Approved)
* Bulk actions
* Filters by source, status, date
* Performance optimized layout for large datasets

5. Article Editor + Device Preview

* Split layout
* Rich text editor
* SEO panel (collapsible)
* Tags & metadata panel
* Auto-save indicator
* Version history
* Device preview toggle:
  Desktop | Tablet | Mobile
* Mobile view simulation inside device frame
* Publish / Save Draft CTA clearly differentiated

6. Review Queue (Workflow Board)

* Kanban layout:
  Pending
  Under Review
  Approved
  Rejected
* Drag and drop interaction
* Reviewer notes
* Activity log
* Audit trail

7. Authentication Screens

* Professional corporate login design
* Email + Password
* Google login
* Microsoft login
* Forgot password
* MFA ready design
* Clean split layout or centered card design

UX Requirements:

* 8px grid system
* Consistent spacing scale
* Enterprise typography hierarchy
* Strong contrast accessibility
* Hover, focus, active states
* Loading skeletons
* Toast notifications
* Confirmation dialogs
* Micro interactions (subtle, not flashy)
* Clear destructive action styling
* Proper empty and error states

Visual Direction:

* Minimal, clean, corporate
* Neutral background
* Subtle accent color (blue or deep indigo)
* Rounded corners (8px)
* Soft shadows
* Sharp iconography
* Professional SaaS aesthetic

The final UI should feel scalable, reliable, secure, and ready for enterprise deployment.
